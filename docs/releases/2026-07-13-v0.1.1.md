## Fix

The public plan schema previously allowed identifiers that the runtime rejected. A real implicit-trigger dry-run exposed this when safe underscore identifiers such as `wave_1` passed structured generation but blocked runtime validation. Version 0.1.1 gives plan, result, and review schemas the same lowercase identifier grammar as runtime validation and permits underscores.
